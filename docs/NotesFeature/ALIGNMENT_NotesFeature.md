# 笔记功能需求对齐文档

## 原始需求
用户希望在AI分析完成后，能够选择将分析结果保存为笔记，并在原文中用下划线标识已分析的内容。四种AI分析类型对应四种不同的下划线样式，点击下划线可以查看之前的分析结果，避免重复分析。

## 项目上下文分析

### 现有项目结构
- **技术栈**: React + TypeScript + Vite + Tailwind CSS
- **核心组件**: 
  - `ReaderPage.tsx`: 主要阅读界面
  - `TextSelectionPopup`: 文本选择弹窗
  - `AIModal`: AI分析模态框
  - `ai-service.ts`: AI服务调用
- **现有功能**: 文本选择、AI分析（四种类型）、结果展示

### 现有AI分析类型
根据代码分析，当前支持四种AI分析类型：
1. 词汇分析
2. 语法分析  
3. 语义分析
4. 文化背景分析

## 需求理解确认

### 核心功能需求
1. **保存笔记功能**
   - 在AI分析结果展示后，提供"保存为笔记"按钮
   - 保存时需要关联原文位置和分析内容
   - 需要标识分析类型

2. **下划线标识系统**
   - 四种AI分析类型对应四种不同的下划线样式
   - 下划线样式需要清晰区分且美观
   - 支持同一文本段落的多种分析类型叠加

3. **笔记查看功能**
   - 点击下划线区域显示对应的分析结果
   - 支持快速预览，无需重新调用AI
   - 可能需要支持编辑和删除笔记

### 技术实现考虑
1. **数据存储**
   - 笔记数据结构设计
   - 本地存储方案（localStorage/IndexedDB）
   - 文本位置定位算法

2. **UI交互设计**
   - 下划线样式设计（颜色、样式、叠加效果）
   - 笔记弹窗/侧边栏设计
   - 保存按钮集成到现有AI模态框

3. **性能优化**
   - 大量笔记的渲染性能
   - 文本匹配算法优化
   - 笔记数据的增量加载

## 边界确认

### 功能范围
- ✅ 保存AI分析结果为笔记
- ✅ 四种下划线样式标识
- ✅ 点击查看历史分析
- ✅ 本地数据存储
- ❓ 笔记的编辑功能
- ❓ 笔记的删除功能
- ❓ 笔记的导出功能
- ❓ 跨文档的笔记同步

### 技术约束
- 必须与现有文本选择系统兼容
- 必须保持现有AI分析流程不变
- 需要考虑不同文档格式的兼容性
- 需要保证页面性能不受影响
