# 文件上传与AI识别功能对齐文档

## 项目上下文分析

### 现有项目架构
- **技术栈**: React + TypeScript + Vite + Tailwind CSS
- **路由**: React Router
- **状态管理**: 无全局状态管理（使用React hooks）
- **存储**: 计划使用IndexedDB
- **UI框架**: Tailwind CSS + 自定义组件

### 现有代码模式
- 组件结构: `src/components/` 按功能分类（auth, layout, ui）
- 页面结构: `src/pages/` 独立页面组件
- 类型定义: `src/types/index.ts` 集中管理
- 工具函数: `src/lib/utils.ts`

### 业务域理解
- **核心功能**: 数字阅读平台，支持EPUB和TXT格式
- **AI功能**: 文本选择后的智能解析和解释
- **用户流程**: 登录 → 上传书籍 → 阅读 → AI辅助理解

## 原始需求分析

### 核心需求
1. **文件上传功能**
   - 支持EPUB和TXT格式
   - 文件大小限制（50MB）
   - 拖拽上传支持
   - 上传进度显示

2. **AI识别功能**
   - 文本选择后弹出AI解析
   - 集成ChatAnywhere API
   - 支持多种解析类型（翻译、解释、总结等）

3. **数据存储**
   - 使用IndexedDB存储书籍内容
   - 缓存AI解析结果

## 边界确认

### 包含范围
- ✅ 文件上传组件实现
- ✅ EPUB和TXT文件解析
- ✅ AI服务集成
- ✅ 文本选择和AI弹窗
- ✅ IndexedDB数据存储
- ✅ 基础错误处理

### 不包含范围

- ❌ 用户账户系统（已有登录页面）
- ❌ 云端同步功能
- ❌ 高级AI功能（如语音合成）

## 技术约束

### 现有约束
- 必须与现有React架构兼容
- 使用TypeScript严格模式
- 遵循现有代码风格和组件模式
- 使用Tailwind CSS样式系统

### 新增约束
- AI API密钥通过.env文件管理
- 文件解析需要支持大文件（流式处理）
- IndexedDB操作需要错误恢复机制

## 疑问澄清

### 已明确的决策
1. **AI服务提供商**: 使用ChatAnywhere API（基于配置文件）
2. **文件格式优先级**: EPUB > TXT（EPUB功能更丰富）
3. **存储策略**: 本地IndexedDB（离线优先）
4. **UI风格**: 保持现有简洁风格，添加必要的交互反馈

### 需要确认的问题
1. **AI API配置**: 需要用户提供ChatAnywhere API密钥
2. **文件大小限制**: 确认50MB是否合适
3. **错误处理策略**: 网络错误、解析错误的用户体验
4. **性能要求**: 大文件上传和解析的性能预期

## 验收标准

### 功能验收
- [ ] 用户可以通过拖拽或点击上传EPUB/TXT文件
- [ ] 上传过程显示进度条和状态
- [ ] 文件解析成功后显示在书架页面
- [ ] 阅读页面可以正常显示文本内容
- [ ] 选择文本后弹出AI解析选项
- [ ] AI解析结果正确显示
- [ ] 数据持久化存储在IndexedDB

### 技术验收
- [ ] 代码通过TypeScript编译
- [ ] 组件可复用且符合现有架构
- [ ] 错误处理完善
- [ ] 性能满足基本要求（<3s响应）

## 风险评估

### 高风险项
- EPUB文件解析复杂度较高
- AI API调用的网络稳定性
- 大文件处理的内存管理

### 缓解策略
- 使用成熟的EPUB解析库
- 实现API调用重试机制
- 采用流式处理和分块加载
