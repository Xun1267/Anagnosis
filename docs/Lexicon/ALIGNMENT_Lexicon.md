# ALIGNMENT_Lexicon.md

## 项目上下文分析

### 项目基本信息
- **项目名称**: Lexicon (阅識)
- **项目类型**: AI驱动的英文原著阅读器
- **开发阶段**: MVP内测版
- **技术架构**: 纯前端应用 (Static Web App)

### 技术栈分析
- **前端框架**: React.js (推荐)
- **文件解析**: 
  - EPUB: Epub.js
  - PDF: PDF.js
  - TXT: 原生JavaScript
- **数据存储**: 浏览器本地存储 (IndexedDB)
- **AI集成**: 直接调用外部API
- **UI特效**: 毛玻璃效果 (backdrop-filter)

### 现有项目约束
- 无后端服务器
- 无用户数据库
- 数据仅存储在本地浏览器
- 使用固定邀请码 "welcome"
- 硬编码API配置 (测试阶段)

## 需求理解确认

### 原始需求
用户提供了完整的PRD文档，要求制作MVP版本，实现:
1. 文件导入与解析 (EPUB、PDF、TXT)
2. AI智能解析功能 (划词/划句)
3. 阅读体验与进度管理
4. 本地数据存储
5. 邀请码准入机制

### 边界确认
**包含功能**:
- 文件导入和解析 (EPUB、TXT优先，PDF可选)
- AI划词解析弹窗
- 目录导航
- 阅读进度保存
- 个性化阅读设置
- 邀请码验证
- 毛玻璃UI效果

**不包含功能**:
- 后端服务器
- 用户注册/登录
- 云端数据同步
- 复杂的笔记系统
- 生词本功能

### 疑问澄清

#### 已明确的技术决策
1. **文件格式优先级**: EPUB > TXT > PDF (PDF解析复杂度高，可作为后续优化)
2. **UI框架选择**: React.js (生态成熟，组件化开发)
3. **状态管理**: 使用React Context或简单的useState (避免过度工程化)
4. **样式方案**: CSS Modules + 毛玻璃效果
5. **构建工具**: Vite (快速开发，现代化构建)

#### 需要确认的设计细节
1. **AI解析弹窗的具体交互**: 点击关闭 vs 点击外部关闭 vs ESC键关闭
2. **文件大小限制**: 是否需要对导入文件设置大小限制
3. **错误处理策略**: AI API调用失败时的用户体验
4. **响应式设计范围**: 是否需要支持移动端

## 风险评估

### 技术风险
1. **EPUB解析复杂度**: 需要处理不同格式的EPUB文件
2. **AI API稳定性**: 外部API可能存在限流或故障
3. **浏览器兼容性**: 毛玻璃效果和IndexedDB支持
4. **文件大小处理**: 大文件可能导致浏览器性能问题

### 缓解策略
1. 使用成熟的epub.js库
2. 实现API调用重试机制和错误提示
3. 提供降级方案和兼容性检测
4. 设置合理的文件大小限制
